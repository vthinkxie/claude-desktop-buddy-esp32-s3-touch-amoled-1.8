# Safe-Area for Rounded AMOLED — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish a single-source-of-truth safe-draw region in the logical 184×224 canvas and migrate all UI call sites to use it, so rounded-bezel corners stop clipping text, icons, and panel edges.

**Architecture:** Add `SAFE_INSET` (single constant, symmetric) plus derived `SAFE_L/T/R/B/W/H` in `src/hw/display.h`. A compile-time `DEBUG_SAFE_BOX` flag draws a 1-px green calibration rectangle in `hwDisplayPush()`. Migration is purely coordinate substitution at existing call sites — no API change to any subsystem.

**Tech Stack:** C++17, Arduino framework 3.x, Arduino_GFX, PlatformIO (pioarduino ESP32-S3), Waveshare 1.8" AMOLED hardware.

**Testing note:** This is embedded firmware with no automated test harness. Verification is manual and on-device — each phase ends with `pio run -e waveshare-amoled -t upload` + a visual check on the board. Treat the "verify" steps as mandatory, not optional.

**Reference spec:** `docs/superpowers/specs/2026-04-19-safe-area-design.md`

## Phase 2 — Migrate Call Sites

Migration rule per call site:
- `setCursor(x_small, ...)` where `x_small <= 8` → use `SAFE_L`.
- `setCursor(W - N, ...)` where N is a small right-margin → use `SAFE_R - <text-width>` (or `SAFE_R - N` if N was already right-aligned).
- `setCursor(..., y_large)` where `y_large` is near `H` → use `SAFE_B - <line-height>`.
- `setCursor(..., y_small)` where `y_small` is near 0 → use `SAFE_T` (or a small offset from it).
- `fillRect(0, TOP, W, H - TOP, ...)` (full-width clears) → keep as-is; background fills are excluded per spec.
- `fillRoundRect(mx, my, mw, mh, ...)` — verify bounds fit inside `[SAFE_L, SAFE_R] × [SAFE_T, SAFE_B]`; adjust if not.

Hardcoded middle-of-canvas coordinates (e.g. `y = 110`, `y = 160`) are NOT migrated — only edge-adjacent anchors.

## Rollback

Every task in Phase 2 is an isolated commit touching text/coordinate literals only. If a screen regresses visually after a migration task, `git revert <task-commit>` undoes exactly that surface. No cross-file coupling — reverts won't cascade.

Phase 1 is additive: the constants are passive until used, and `DEBUG_SAFE_BOX` is flag-gated. Reverting Phase 1 leaves no trace.

## Out of Scope (per spec)

- Hardware-layer alert bar (`src/hw/display.cpp:71-77`). Operates in physical 368×448 coords with its own inset; not affected.
- Full-canvas background fills (`fillRect(0, 0, HW_W, HW_H)` / `fillScreen`). Bezel hiding corner pixels of a solid fill is invisible to the user.
- Buddy body pixel drawing (only the anchor migrates, which in this plan turned out to be a no-op after audit).
- Rounded-corner pixel masking of the canvas (see Non-Goals in spec).
